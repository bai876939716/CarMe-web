# CarMe-web 前端项目结构说明

> CarMe 汽车改装可视化平台前端，基于 React + Vite + TypeScript 构建。

---

## 一、项目根目录结构

```
CarMe-web/
├── index.html          # 应用入口 HTML
├── package.json        # 依赖与脚本配置
├── package-lock.json   # 依赖版本锁定
├── vite.config.ts      # Vite 构建配置
├── tsconfig.json       # TypeScript 根配置（引用子配置）
├── tsconfig.app.json   # 应用源码 TS 配置
├── tsconfig.node.json  # Node 环境 TS 配置
├── eslint.config.js    # ESLint 配置
├── .gitignore          # Git 忽略规则
├── README.md           # 项目说明
├── public/             # 静态资源（不参与打包）
│   └── vite.svg
└── src/                # 源代码目录 ⬇️
```

---

## 二、根目录文件说明

| 文件 | 作用 |
|------|------|
| **index.html** | SPA 入口，挂载 `#root`，通过 `<script type="module">` 引入 `src/main.tsx` |
| **package.json** | 项目元信息、依赖（React、Ant Design、Three.js、Zustand 等）、npm 脚本（`dev` / `build` / `lint` / `preview`） |
| **vite.config.ts** | Vite 配置：React 插件、开发服务器代理 `/api` 和 `/models` 到后端 `localhost:8080` |
| **tsconfig.json** | 引用 `tsconfig.app.json` 与 `tsconfig.node.json` 的项目引用配置 |
| **tsconfig.app.json** | 应用源码的 TS 配置：目标 ES2022、严格模式、JSX、只覆盖 `src` |
| **tsconfig.node.json** | Vite 等 Node 脚本的 TS 配置 |
| **eslint.config.js** | ESLint 9 扁平配置：TypeScript、React Hooks、React Refresh 规则 |
| **public/** | 静态资源目录，内容会直接复制到构建输出根目录 |

---

## 三、src 目录结构概览

```
src/
├── main.tsx            # React 应用入口
├── App.tsx             # 根组件（路由、主题、布局）
├── App.css             # App 组件样式
├── index.css           # 全局样式
├── api/                # 接口层
├── assets/             # 需打包的静态资源
├── components/         # 通用/业务组件
├── mock/               # 模拟数据
├── pages/              # 页面级组件

├── stores/             # 状态管理（Zustand）

```

## 四、src 各目录与文件说明

### 4.1 入口与根组件

| 路径 | 作用 |
|------|------|
| **main.tsx** | 引入 Ant Design 重置样式、全局样式，挂载 React 应用到 `#root` |
| **App.tsx** | 配置 Ant Design 深色主题、路由（BrowserRouter），使用 `AppLayout` 包裹子路由；定义首页、改装工坊、商城、社区、个人中心等路由 |
| **App.css** | App 组件相关样式 |
| **index.css** | 全局样式（如 `body`、`#root` 等） |

### 4.2 api/ — 接口层

| 路径 | 作用 |
|------|------|
| **api/client.ts** | 封装 Axios 实例，baseURL 为 `/api`；定义 `api` 对象，包含：<br>• 车型：`getCars`<br>• 配件：`getParts`、`getHotParts`<br>• 方案：`saveWork`、`getMyWorks`、`getCommunityWorks`、`toggleLike`、`toggleFavorite`<br>统一处理后端 `ApiResponse<T>` 和 `PageResult<T>` 结构 |

### 4.3 assets/ — 静态资源

| 路径 | 作用 |
|------|------|
| **assets/react.svg** | React 相关图标，可被 import 引用并参与 Vite 打包 |

### 4.4 components/ — 组件

| 路径 | 作用 |
|------|------|
| **components/layout/AppLayout.tsx** | 主布局：顶部导航栏（Logo、首页/改装工坊/商城/社区/个人中心）、登录/注册按钮、`<Outlet />` 渲染子路由 |
| **components/CarSelector.tsx** | 车型选择器：从 `workshopStore` 拉取车型列表，用 Ant Design `Select` 选择车型 |
| **components/PartPanel.tsx** | 配件选择面板：轮毂、颜色选择，调用 `loadWheels` / `loadColors`，支持保存改装方案 |
| **components/CarModel3D.tsx** | 3D 车型展示：基于 `@react-three/fiber`，用占位几何体（车身 box、轮毂 cylinder）展示当前选中车型、轮毂、颜色；可接入真实 3D 模型 |

### 4.5 mock/ — 模拟数据

| 路径 | 作用 |
|------|------|
| **mock/data.ts** | 首页、社区、商城等使用的静态数据：<br>• `mockCommunityPosts` — 社区帖子<br>• `mockProducts` — 商品<br>• `mockHotWorks` — 热门改装<br>• `mockActivities` — 活动<br>• `mockUser`、`mockMyCars`、`mockMyWorks` — 用户与方案模拟数据 |

### 4.6 pages/ — 页面

| 路径 | 作用 |
|------|------|
| **pages/HomePage.tsx** | 首页：Hero、热门改装、社区精选、最新配件、官方活动、页脚 |
| **pages/WorkshopPage.tsx** | 改装工坊：车型选择 → 配件选择 → 3D 预览，与 `workshopStore`、`WorkshopScene` 联动 |
| **pages/MallPage.tsx** | 配件商城页 |
| **pages/CommunityPage.tsx** | 社区页 |
| **pages/ProfilePage.tsx** | 个人中心页 |

### 4.7 scenes/ — 3D 场景

| 路径 | 作用 |
|------|------|
| **scenes/WorkshopScene.tsx** | 改装工坊 3D 场景：`@react-three/fiber` 的 `Canvas`，`PerspectiveCamera`、`OrbitControls`、`Environment`、`CarModel3D` 等，支持旋转/缩放/平移查看 |

### 4.8 stores/ — 状态管理

| 路径 | 作用 |
|------|------|
| **stores/workshopStore.ts** | 改装工坊状态（Zustand）：<br>• 状态：`selectedCar`、`selectedWheel`、`selectedColor`、`cars`、`wheels`、`colors`、`works`<br>• 动作：`setSelectedCar`、`setSelectedWheel`、`setSelectedColor`、`loadCars`、`loadWheels`、`loadColors`、`saveWork`、`loadMyWorks` |

### 4.9 types/ — 类型定义

| 路径 | 作用 |
|------|------|
| **types/index.ts** | 全局 TS 类型：<br>• `CarModel` — 车型<br>• `Part` — 配件<br>• `Work` — 改装方案<br>• `CommunityPost` — 社区帖子<br>• `Product` — 商品<br>• `UserProfile` — 用户资料 |

## 六、主要数据流

1. **首页**：使用 `mock/data.ts` 的 mock 数据渲染。
2. **改装工坊**：`WorkshopPage` → `workshopStore`（Zustand）→ `api/client` 请求后端 `/api/cars`、`/api/parts`、`/api/works`。
3. **3D 预览**：`WorkshopScene` 使用 `CarModel3D`，读取 `selectedCar`、`selectedWheel`、`selectedColor` 实时展示（当前为占位几何体）。
4. **代理**：开发环境下 `/api`、`/models` 经 Vite 代理转发到 `http://localhost:8080`。
